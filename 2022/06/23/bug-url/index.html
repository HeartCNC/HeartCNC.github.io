URL链接获取search参数是undefined？hash丢失了？记一下一些坑

2022年6月23日
  • url
  • bug

URL链接取参数问题，search和hash隐藏的坑

search​

  • 场景一

http://localhost:8083/

取得参数 channel_code 的类型是 undefined

note

unescape 方法会将 undefined 转为字符串 'undefined'

http://localhost:8083/?channel_code

这里要根据具体取参的方法判定了吧，channel_code 可能是 undefined 或 'undefined'

  • 场景二

http://localhost:8083/?channel_code=

取参数 channel_code 的类型是字符串 ''

  • 场景三

http://localhost:8083/?channel_code=1

取参数 channel_code 的类型是字符串 '1'

hash​

看看这个 https://stackoverflow.com/questions/5315396/bug-window-location-href-hash-in-safari

就是说IOS可能会发生hash丢失的情况，但是具体什么时候，谁也不知道

那么search的参数还是最好不要拼在hash后面，不然可能发生丢失
